# KẾ HOẠCH KIỂM THỬ TÍCH HỢP CHỐNG GIAN LẬN (INTEGRATION TEST PLAN)

Tài liệu này đặc tả các kịch bản kiểm thử tích hợp (Integration Test Scenarios) nhằm xác minh độ chính xác của 3 lớp phòng vệ chống gian lận và cơ chế dự phòng của hệ thống **AFAS** trên môi trường tích hợp thực tế.

---

## 📋 DANH SÁCH CÁC KỊCH BẢN KIỂM THỬ TÍCH HỢP (TEST SCENARIOS)

| Mã số Test | Tên Kịch bản kiểm thử | Tầng Xác thực | Điều kiện đầu vào (Inputs) | Các bước thực hiện (Steps) | Kết quả mong đợi (Expected Results) |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **IT-001** | **Điểm danh hợp lệ thành công** *(Normal Flow)* | Tích hợp toàn phần | - MSSV: `SE170123`<br>- QR Token: Còn hạn 5s<br>- GPS: Khớp phòng (`d = 5m`) <br>- Device UUID: Đúng máy chính chủ<br>- Mạng: Wi-Fi trường | 1. Sinh viên bấm "Scan QR Check-in" trên App di động.<br>2. Xác thực Face ID di động thành công.<br>3. Quét mã QR, thu thập GPS, Wi-Fi IP và UUID.<br>4. Gửi request điểm danh lên Server. | - Server ghi nhận thành công bản ghi điểm danh trạng thái `Present`.<br>- Màn hình giảng viên WebSocket sáng xanh tên SV.<br>- Ảnh selfie Face ID bị xóa ngay khỏi Server. |
| **IT-002** | **Chặn chụp ảnh QR chia sẻ từ xa** *(Layer 1)* | Lớp 1 (QR Token) | - MSSV: `SE170123`<br>- QR Token: Quá hạn 30s<br>- GPS: Khớp phòng (`d = 5m`) | 1. Sinh viên quét ảnh QR chụp gửi từ xa.<br>2. App gửi token đã hết hạn lên Server.<br>3. Server đối khớp Token với phiên hiện hành. | - Server từ chối ghi nhận điểm danh.<br>- Trả về lỗi: **"QR code expired"**.<br>- Không tạo bản ghi đi học trong DB. |
| **IT-003** | **Chặn điểm danh hộ khi ở nhà** *(Layer 2)* | Lớp 2 (Geofencing) | - MSSV: `SE170123`<br>- QR Token: Hợp lệ<br>- GPS: Ở nhà (`d = 5km`) | 1. Sinh viên nhờ bạn đọc mã PIN hoặc dùng Fake GPS.<br>2. App gửi tọa độ GPS thực tại nhà lên Server.<br>3. Server chạy công thức Haversine đo khoảng cách. | - Server từ chối ghi nhận điểm danh.<br>- Tạo bản ghi `AttendanceRecord` với trạng thái `Fraud_Declined`.<br>- Trả về lỗi: **"Out of allowed radius"**. |
| **IT-004** | **Chặn điểm danh hộ trên máy lạ** *(Layer 3)* | Lớp 3 (Device Binding) | - MSSV: `SE170123`<br>- QR Token: Hợp lệ<br>- GPS: Khớp phòng<br>- Device UUID: Lạ | 1. Sinh viên đi học đăng nhập tài khoản của bạn bè vào máy của mình để quét mã điểm danh hộ.<br>2. App gửi Device UUID lạ của máy lên Server. | - Server từ chối ghi nhận điểm danh.<br>- Trả về lỗi: **"Device UUID Mismatch"**.<br>- Ghi lịch sử cảnh báo đăng nhập trái phép vào bảng SystemLog. |
| **IT-005** | **Check mạng Wi-Fi nội bộ trường** *(Network check)* | Bảo mật mạng | - MSSV: `SE170123`<br>- QR Token: Hợp lệ<br>- GPS: Khớp phòng<br>- Mạng: Dùng mạng di động 4G | 1. Sinh viên tắt Wi-Fi trường và sử dụng 4G cá nhân để điểm danh.<br>2. App gửi địa chỉ IP Gateway 4G lên Server.<br>3. Server đối khớp với dải IP Gateway Wi-Fi trường. | - Server cảnh báo mức độ tin cậy thấp hoặc từ chối tùy thuộc vào cấu hình chặt chẽ của giảng viên.<br>- Trả về yêu cầu: **"Vui lòng kết nối Wi-Fi trường để điểm danh."** |
| **IT-006** | **Dự phòng mã PIN khi camera lỗi** *(Fallback 2)* | Luồng dự phòng | - MSSV: `SE170123`<br>- Mã PIN: Khớp mã đổi sau 30s<br>- GPS: Khớp phòng | 1. Sinh viên bị hỏng camera di động, chọn "Điểm danh bằng mã PIN".<br>2. Nhập mã PIN 6 số hiển thị trên máy chiếu.<br>3. App gửi mã PIN kèm định vị GPS lên Server. | - Server đối khớp mã PIN còn hạn sử dụng.<br>- Thực hiện tiếp các bước check Geofencing và Device UUID.<br>- Ghi nhận điểm danh thành công trạng thái `Present`. |
